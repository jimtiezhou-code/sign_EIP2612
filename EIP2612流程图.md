# EIP-2612 签名存款流程图

## 总览

```
用户输入金额 → 点击「签名存款」→ 钱包弹出签名确认 → 用户签名（免费）
                                                         │
                                                         ▼
                                                  钱包生成 v, r, s
                                                         │
                                                         ▼
                                          调用 TokenBank.permitDeposit()
                                                         │
                                                         ▼
                                              链上一步完成：授权 + 存款
```

## 二、EIP-712 签名构造（链下 / 钱包内部）

```
                        ┌─────────────────────────────────────────┐
                        │          EIP-712 Domain 哈希             │
                        │                                         │
  TYPE_HASH             │  (合约常量，部署时写入)                   │
  = keccak256(          │                                         │
    "EIP712Domain(      │  bytes32 TYPE_HASH                      │
     string name,       │  = keccak256(                           │
     string version,    │    "EIP712Domain(string name,           │
     uint256 chainId,   │      string version,uint256 chainId,    │
     address verifying  │      address verifyingContract)"        │
     Contract)"         │    )                                    │
  )                     │                                         │
                        │  domainSeparator                        │
  _hashedName           │  = keccak256(                           │
  = keccak256(          │    abi.encode(                          │
    "BaseERC20"         │      TYPE_HASH,                         │
  )                     │      _hashedName,   ─── keccak256("BaseERC20")
                        │      _hashedVersion,─── keccak256("1")
  _hashedVersion        │      block.chainid, ─── 31337
  = keccak256(          │      address(this)  ─── token合约地址
    "1"                 │    )                                    │
  )                     │  )                                     │
                        └─────────────────────────────────────────┘

                        ┌─────────────────────────────────────────┐
                        │          Permit Struct 哈希              │
                        │                                         │
  PERMIT_TYPEHASH       │  (合约常量)                              │
  = keccak256(          │                                         │
    "Permit(            │  bytes32 PERMIT_TYPEHASH                │
     address owner,     │  = keccak256(                           │
     address spender,   │    "Permit(address owner,               │
     uint256 value,     │      address spender,                   │
     uint256 nonce,     │      uint256 value,                     │
     uint256 deadline)" │      uint256 nonce,                     │
  )                     │      uint256 deadline)"                 │
                        │    )                                    │
  实际参数：             │                                         │
  owner    = 用户地址    │  structHash                             │
  spender  = TokenBank  │  = keccak256(                           │
  value    = 100*1e18   │    abi.encode(                          │
  nonce    = 0          │      PERMIT_TYPEHASH,                   │
  deadline = now+3600   │      owner,                             │
                        │      spender,                           │
                        │      value,                             │
                        │      nonce,   ─── 从合约读取,每次+1      │
                        │      deadline ─── 过期时间戳             │
                        │    )                                    │
                        │  )                                     │
                        └─────────────────────────────────────────┘

                                  │                  │
                                  ▼                  ▼
                           domainSeparator      structHash
                                  │                  │
                                  └────────┬─────────┘
                                           ▼
                                 ┌───────────────────┐
                                 │  最终摘要 digest    │
                                 │                   │
                                 │  digest =          │
                                 │  keccak256(        │
                                 │    "\x19\x01"      │ ← EIP-191 前缀
                                 │    || domainSeparator
                                 │    || structHash   │
                                 │  )                 │
                                 └───────┬───────────┘
                                         │
                                         ▼
                                 ┌───────────────────┐
                                 │  ECDSA 签名        │
                                 │                   │
                                 │  (v, r, s) =      │
                                 │  secp256k1_sign(  │
                                 │    privateKey,    │ ← 用户私钥
                                 │    digest         │
                                 │  )                │
                                 └───────┬───────────┘
                                         │
                                         ▼
                                  signature (65 bytes)
                                  r: bytes32 (前32字节)
                                  s: bytes32 (中32字节)
                                  v: uint8   (末1字节, 27或28)
```

## 三、链上验证与执行

```
  TokenBank.permitDeposit(owner, amount, deadline, v, r, s)
  │
  │  msg.sender 可以是任何人（不一定是 owner）
  │  owner 的授权意愿由签名 v,r,s 证明
  │
  ├─ require(amount > 0)
  │
  ├─ IERC20Permit(token).permit(owner, address(this), amount, deadline, v, r, s)
  │  │
  │  │  ┌──────────────────────────────────────────────────────┐
  │  │  │            ERC20Permit.permit()                       │
  │  │  │                                                      │
  │  │  │  ① require(block.timestamp <= deadline)              │
  │  │  │     │ 否则 → revert ERC2612ExpiredSignature          │
  │  │  │     │                                                │
  │  │  │  ② _useNonce(owner) → 返回当前 nonce, 然后自身 +1    │
  │  │  │     │ 保证同一签名只能用一次                          │
  │  │  │     │                                                │
  │  │  │  ③ 重算 structHash (与链下签名完全一致)               │
  │  │  │     structHash = keccak256(                           │
  │  │  │       abi.encode(                                    │
  │  │  │         PERMIT_TYPEHASH,  ← 合约常量                  │
  │  │  │         owner,                                       │
  │  │  │         spender,          ← address(this) = TokenBank │
  │  │  │         value,                                       │
  │  │  │         nonce,                                       │
  │  │  │         deadline                                     │
  │  │  │       )                                              │
  │  │  │     )                                                │
  │  │  │                                                      │
  │  │  │  ④ 重算 digest (与链下签名完全一致)                   │
  │  │  │     digest = _hashTypedDataV4(structHash)            │
  │  │  │            = keccak256(                              │
  │  │  │                "\x19\x01"                            │
  │  │  │                || _domainSeparatorV4()               │
  │  │  │                || structHash                         │
  │  │  │              )                                       │
  │  │  │                                                      │
  │  │  │  ⑤ ecrecover(digest, v, r, s) → signer 地址          │
  │  │  │                                                      │
  │  │  │  ⑥ require(signer == owner)                          │
  │  │  │     │ 否则 → revert ERC2612InvalidSigner             │
  │  │  │     │                                                │
  │  │  │  ⑦ _approve(owner, spender, value)                   │
  │  │  │     allowances[owner][spender] = value               │
  │  │  │     emit Approval(owner, spender, value)             │
  │  │  └──────────────────────────────────────────────────────┘
  │  │
  │  └─ permit 成功返回
  │
  ├─ token.transferFrom(owner, address(this), amount)
  │  │  TokenBank 已经有了 owner 的授权额度，直接划转
  │  └─ 成功
  │
  ├─ depositBalances[owner] += amount
  │
  └─ emit Deposit(owner, amount)
```

## 四、签名 vs 传统授权 对比

```
  传统 approve + deposit (两笔交易)        permitDeposit (一笔交易)
  ════════════════════════════════        ════════════════════════

  用户                               用户
  │                                  │
  ├─ tx1: approve(TokenBank, 100)    ├─ 钱包签名 (离线, 免费)
  │   gas: ~45,000                   │    │
  │   等待确认 (~15s)                 │    │ 签名内容:
  │                                  │    │ "我(owner)授权TokenBank
  ├─ tx2: deposit(100)               │    │  使用100个token,
  │   gas: ~80,000                   │    │  nonce=0, 1小时后过期"
  │   等待确认 (~15s)                 │    │
  │                                  │    │ ECDSA_sign → v, r, s
  完成                               │    │
                                     │    ▼
  总gas: ~125,000                    ├─ tx1: permitDeposit(
  总时间: ~30s                       │          owner, 100, deadline,
  钱包交互: 2次确认交易               │          v, r, s)
                                     │   gas: ~100,000
                                     │   等待确认 (~15s)
                                     │
                                     完成

                                    总gas: ~100,000 (~20%节省)
                                    总时间: ~15s
                                    钱包交互: 1次签名 + 1次确认交易
```

## 五、安全机制

```
  ┌──────────────┬────────────────────────────────────────────────────┐
  │   机制        │   防护目标                                           │
  ├──────────────┼────────────────────────────────────────────────────┤
  │              │                                                    │
  │  nonce        │  防重放攻击                                         │
  │  (每次+1)     │  同一签名第2次使用 → nonce已变 → structHash不同      │
  │              │  → ecrecover 恢复出的 signer 不对 → revert            │
  │              │                                                    │
  ├──────────────┼────────────────────────────────────────────────────┤
  │              │                                                    │
  │  deadline     │  防签名长期滞留                                     │
  │              │  如果签名被截获但迟迟未上链,过了 deadline 就失效       │
  │              │                                                    │
  ├──────────────┼────────────────────────────────────────────────────┤
  │              │                                                    │
  │  signer       │  防伪造授权                                         │
  │  == owner    │  A 不能拿 A 自己签的名去花 B 的 token                 │
  │              │                                                    │
  ├──────────────┼────────────────────────────────────────────────────┤
  │              │                                                    │
  │  domainSep-   │  防跨链/跨合约重放                                  │
  │  arator 包含  │  同一签名在 Ethereum 上有效,在 Polygon 上就无效      │
  │  chainId +    │  因为 chainId 不同 → domainSeparator 不同            │
  │  verifying-   │  同理跨合约也无效                                   │
  │  Contract     │                                                    │
  │              │                                                    │
  ├──────────────┼────────────────────────────────────────────────────┤
  │              │                                                    │
  │  \x19\x01    │  防签名数据被误解析为交易                             │
  │  前缀         │  EIP-191 规范: \x19 前缀标记这不是普通交易            │
  │              │  \x01 标识是 EIP-712 类型化数据                      │
  │              │                                                    │
  └──────────────┴────────────────────────────────────────────────────┘
```
